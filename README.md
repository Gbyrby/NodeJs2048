# Procces of game
- Пользователь заходит на сайт /
    - Если у него есть куки которые все еще действуют
        - сделать синхронизацию (выдается доска,очки, количество выполненных шагов)
        - обновить время жизни куки
    - Если у него нет куки\куки уже считается не действующим
        - создать никнейм 
        - создать UUID для его идентификации
        - задать куки
        - выдать доску
- Считывая управление игрока
    - запросить случайно добавленный блок
    - сделать расчет игры самостоятельно
    - сделать запрос на сервер
    - если данные не совпадают сделать синхронизацию /sync
        - (выдается доска,очки, количество выполненных шагов)
    - если игра у обоих завершилась и количество очков выше чем у самого слабого игрока в топ 10, сохранить в БД. забыть куки игрока
- раз в час проверять время жизни куки у всех пользователей, не живые сессии удалять из памяти

- Игровая сессия должна сохранять:
    - UUID пользователя
    - никнейм
    - дату последнего обновления куки
    - доску
    - количество ходов
    - количество очков
